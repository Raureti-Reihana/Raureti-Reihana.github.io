I had a problem where I couldn't get through one of the Javascript kata challenges. I ran the test for the javascript-kata challenges and then read the error messages. I also used the test.js files to see what exactly the challenge was looking for. Reflect on how confident you feel using each of these problem-solving techniques/processes:

There are a few ways we can increase our neuroplasticity, and they are in my opinion the basics to living a healthy lifestyle. Things we might do to increase neuroplasticity are:

  • Pseudocode

    • While doing the javascript-kata challenges i felt like the way they had the instructions was already similar to pseudocode, something perhaps i could have done is to try and simplify those instructions even more and break them down.

  • Trying something

    • This is probably my most used technique, just because I like to see how much of the challenge I can do on my own before I need to ask for help or do more research into the task. I often try something to see if it works and if it doesn’t then I will try changing something else again. The downside to this is that the code can get messy and then I have a bunch of missed attempts all mixed into one.

  • Rubber ducky method

    • Didn’t use this method because I honestly don’t know enough about this part of JS and I can't come up with any answers without seeking help from somewhere, but I will get there.

  • Reading error messages

    • This was very helpful because it would give you a clue and I would be able to think to myself, “this is what it expects the answer to be and how can I get to that point”. It also gives you a point where you can almost work backwards to find the how in between.

  • Console.logging

    • I do like this technique but am unsure of how to console.log out when i’m using JS in vscode. I will find out, because this is really handy to see what bits of codes are currently outputting

  • Googling

    • I did a lot of googling, especially when trying to figure out the terms and jargon. It was especially useful when I was learning about different array methods.

  • Asking coaches for help

    • Have had to do this a couple of times because I get to a spot and I don't know what to do from there.

  • Improving your process with reflection

    • I have noticed the amount of information that I have been able to retain during this course with all the reflections and can see why they want us to do it so often. It is a very powerful tool and something I should implement into my life outside of Dev Academy as well.
