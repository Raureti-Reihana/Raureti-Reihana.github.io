What is Javascript?

JavaScript is the Programming Language that can update and change both HTML and CSS. It gives interactivity and functionality to websites.
When I'm explaining the relationship between HTML, CSS and Javascript I always use the analogy of a house which makes it easier for me to remember and explain.
So let's imagine that we have our house which starts off with our frames. We can liken this to HTML as it is the bones and the foundation to our house and also gives us our structure to build upon. Just like the frames of our house, html is quite boring by itself and doesn’t show much character, that's where CSS comes in.
We can pretend that CSS is all the things that give our house its character whether it's the color of paint on the wall, the type of cladding or the positioning of the windows. CSS can be as boring or as out there as you like the choice is yours.
Finally Javascript is similar to all the interactive features that we have in our house like turning on and off light switches or water taps, closing and opening doors or even house rules e.g “if you make a mess then you clean it up!!”
So HTML gives us our foundation and structure, CSS gives us our design and styling and Javascript gives us functionality and protocol.

Control Flow, Conditionals and Loops

Control Flow is the order in how the computer reads code. Typically it reads from top to bottom.
Here’s an analogy to help you understand. Let’s imagine you are baking a cake and we have a recipe/method. While we are baking our cake we follow our method from start to finish, top to bottom. Imagine trying to make a cake from the bottom up, it just wouldn’t work.
Conditionals can be if statements and if else statements which simply put are, if this is doing this then do this or else do this. Similar to following our recipe for our cake, if our cake mixture is too dry then add milk or else keep mixing.
Loops have different types but can be simply used and thought of as, continue to repeat this step until this condition is met. So if we use our cake making analogy again we could say, keep mixing and adding milk until the desired consistency is met.

What is the DOM?

The DOM or Document Object Model is a programming interface or a representation of all the elements in a web page document that is able to be manipulated with Javascript.

Document refers to the webpage.

Object refers to the elements of that web page e.g p tags, body tags and img tags.

Model refers to the relationship between each element.

The DOM can be imagined as a dynamic view of an HTML file. If we think of it in a tree arrangement like this

If we want to interact with the DOM we can do things like access elements, create new elements, remove elements, modify content of elements and respond to events.
